# Спецификация текстовой защиты UX-аудита: Travel X

Дата: 2026-05-05

## Цель
Подготовить текстовую защиту UX-аудита для проекта «Travel X» (бронирование космических туров) с доказательной аргументацией эффективности дизайна. Никакого кода — только вербальные доказательства и расчет времени выполнения задач.

## Источники доказательств
Используются 5 экранов интерфейса, предоставленные заказчиком:
1. Главная страница (hero + CTA, верхняя навигация, плавающий AI-бот)
2. Направления (фильтры, карточки туров, CTA «Подробнее», плавающий AI-бот)
3. Регистрация (форма, подсказки форматов, CTA)
4. Вход (форма, CTA)
5. Личный кабинет (успешный вход, карточки профиля, загрузка фото, CTA, плавающий AI-бот)

Подтвержденные сценарии:
- «Бронирование в один клик» = нажатие CTA «Выбрать маршрут» на главной или CTA в карточках туров.
- «AI-бот» = плавающая кнопка «AI Ассистент» в правом нижнем углу.

## Методика Time-on-Task (для законов Хика и Фиттса)
- Условия: desktop + мышь, спокойная среда, пользователь без ошибок и повторов.
- Декомпозиция: ориентация → поиск элемента → принятие решения → наведение/клик → подтверждение результата.
- Константы времени:
  - ориентация/скан: 0.6–1.0 с
  - выбор среди 2–4 опций: 0.6–1.2 с
  - наведение на крупный CTA/плавающую кнопку: 0.3–0.5 с
  - клик: 0.1–0.2 с
  - ввод текста: 0.25–0.35 с на символ

## Формат ответа
1) Сначала Markdown-таблица Time-on-Task для 4 сценариев:
   - Регистрация нового аккаунта
   - Выбор тура (фильтрация + «Подробнее»)
   - Бронирование в один клик
   - Вызов AI-бота
2) Далее разбор 4 законов:
   - Закон Хика
   - Закон Фиттса
   - Закон Якоба
   - Закон Миллера
3) Далее 10 эвристик Нильсена (каждая отдельно).

## Обязательная фраза-доказательство
Каждый аргумент должен содержать фразу:
- «там в этой странице есть …» ИЛИ
- «там на этой странице есть …»

После фразы — указание конкретного элемента интерфейса, подтверждающего закон/эвристику.

## Тон
Строго-академический, доказательный, без кода, без лишних оценок.

## Структура блоков
### Блок 1. Time-on-Task (таблица)
- Обязательная посекундная разбивка по микро-шагам.
- Итоги показывают быстрые сценарии за счет крупных целей и ограниченного количества решений.

### Блок 2. Законы UX
- 4 закона, по 1–2 доказательства на закон.
- Каждый аргумент строго с фразой «там на/в этой странице есть …».

### Блок 3. Эвристики Нильсена
- 10 пунктов, по 1 доказательству на каждый.
- Каждый аргумент строго с фразой «там на/в этой странице есть …».

## Готовность к защите
Материал рассчитан на устное выступление: сначала цифры (скорость), затем доказательства по законам и эвристикам.
