# Travel X на Django

`Travel X` теперь работает как Django-проект с базой данных, системой регистрации и входа, личным кабинетом, фотографиями профиля и перенесенной публичной частью из старой версии сайта.

## Что есть в проекте

- Публичная главная страница в стиле старого `Travel X`.
- Страница направлений с карточками туров, психологическим тестом и модальным бронированием.
- Страница успешной регистрации.
- Страницы регистрации, входа и личного кабинета пользователя.
- Загрузка фотографии профиля с проверкой формата изображения.
- Фото по умолчанию, если пользователь еще не загрузил свой аватар.
- Хранение фотографий в папках пользователей вида `media/profile_photos/user_<id>/...`.
- Хранение пути к фотографии в базе данных SQLite.
- Закрытая конфиденциальная страница только для администратора.
- Django admin для управления пользователями и профилями.

## Основные маршруты

- `/` — публичная главная страница для обычных пользователей.
- `/destinations/` — страница космических направлений.
- `/register/` — регистрация нового пользователя.
- `/login/` — вход в аккаунт.
- `/success/` — страница успешной регистрации после создания аккаунта.
- `/dashboard/` — личный кабинет пользователя.
- `/staff-home/` — конфиденциальная страница только для администратора.
- `/admin/` — стандартная админ-панель Django.

## Как теперь устроен проект

### Backend

- Django 5.
- SQLite как основная база данных.
- Встроенная модель `User` используется для авторизации.
- Дополнительная модель `UserProfile` хранит:
  - возраст;
  - фотографию профиля;
  - дату создания и обновления профиля.

### Frontend

- Публичные страницы используют дизайн из `oldVersion`, но уже работают через Django templates.
- Закрытые страницы аккаунта используют отдельный современный интерфейс.
- В правом верхнем углу сайта после входа отображаются:
  - имя пользователя;
  - аватар пользователя;
  - ссылка на кабинет;
  - ссылка на закрытую страницу, если пользователь является администратором.

## Структура важных файлов

- `travelx_project/settings.py` — настройки Django, статика, медиа, шаблоны.
- `travelx_project/urls.py` — корневые маршруты проекта.
- `accounts/models.py` — модель профиля пользователя.
- `accounts/views.py` — логика страниц, регистрации, входа и закрытой страницы администратора.
- `accounts/forms.py` — формы регистрации, входа и обновления фото.
- `accounts/admin.py` — регистрация модели профиля в админке.
- `accounts/context_processors.py` — данные пользователя и аватара для навигации.
- `templates/public_base.html` — базовый шаблон публичной части сайта.
- `templates/base.html` — базовый шаблон внутренних страниц и кабинета.
- `templates/accounts/` — все Django-шаблоны страниц.
- `static/css/public_site.css` — стили публичной части, перенесенные из старой версии.
- `static/js/public_site.js` — логика публичных страниц.
- `static/css/styles.css` — стили кабинета и внутренних страниц.
- `static/images/default-avatar.svg` — аватар по умолчанию.

## Работа с фотографиями профиля

- Поддерживаются только изображения: `jpg`, `jpeg`, `png`, `webp`, `gif`.
- При загрузке файл проверяется как изображение.
- Файл сохраняется не в одну общую папку, а в отдельную папку конкретного пользователя.
- В базе данных сохраняется путь к изображению.
- Пользователь видит только свою фотографию в своем кабинете и в шапке сайта.

## Запуск проекта

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Выполнить миграции:

```bash
python manage.py migrate
```

3. При необходимости создать суперпользователя:

```bash
python manage.py createsuperuser
```

4. Запустить сервер:

```bash
python manage.py runserver
```

## Проверка

Для проекта добавлены Django-тесты на:

- загрузку главной и страницы направлений;
- регистрацию пользователя и создание профиля;
- обновление фотографии профиля;
- доступ администратора к закрытой странице;
- запрет доступа обычному пользователю к конфиденциальной странице.
